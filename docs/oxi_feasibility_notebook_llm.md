# Oxi プロジェクト フィージビリティ概要
## Microsoft Office互換エンジンの開発・無料配布事業

---

# 1. 一言でいうと

**「Microsoft Officeと99%互換のソフトを無料で配り、サポートで稼ぐ」**

Red HatがLinuxで年間6,000億円稼いでいるのと同じ構造です。
Officeの対象市場はLinuxより遥かに大きく、年間7.5兆円規模です。

---

# 2. なぜ今まで誰もできなかったか

WordやExcelで作った文書を、無料ソフト（LibreOffice等）で開くと**レイアウトが崩れます**。日本語、Hindi、Tamil等の文書は特にひどい。

これは30年間解決されていない問題です。原因はシンプルで、**Wordが文書をどう表示しているかの計算方法が非公開**だからです。

LibreOfficeは30年かけて独自の計算方法を作りましたが、Wordと違う方法なので、結局ずれる。これが根本原因です。

---

# 3. なぜ我々にはできるのか

**AI（Claude Code）を活用した独自の仕様解析手法**を開発しました。

Microsoft WordのCOM API（公式に提供されているプログラム制御インターフェース）を使い、大量の文書を処理させ、AIがその入出力パターンから**Wordの正確な計算ルールを特定**しました。

具体的には：
- 行の高さの計算方法
- 表のセル幅の自動決定ルール
- 日本語の行末処理（禁則処理）
- 文字の均等配置のアルゴリズム

など、10以上の核心的な計算ルールを全て特定済みです。

**この手法は他のソフトウェアにも応用可能です。** Adobe Illustrator、Photoshop、AutoCAD等、非公開の表示アルゴリズムを持つソフトウェアの互換エンジンを同じ方法で開発できます。

---

# 4. 現在の開発状況

既にPoC（概念実証）が動作しており、Word文書をブラウザ上で表示できます。

**品質指標（SSIM = 画像類似度、1.0が完全一致）：**

| 文書タイプ | 現在のスコア | 目標（2ヶ月後） |
|-----------|-------------|----------------|
| 単純なテーブル | 1.00（完全一致） | - |
| 英語ビジネス文書 | 0.90 | 0.97 |
| 日本語公文書 | 0.70-0.96 | 0.95 |
| 全体平均（156ファイル） | **0.84** | **0.95+** |

テスト方法：同じ文書をWordとOxiで表示し、画面をピクセル単位で比較しています。

# 6. なぜ今がチャンスか

## 6.1 各国政府の「脱Microsoft」の動き

- **日本**: デジタル庁がOSS推進。自治体のOffice依存が課題
- **インド**: Digital India政策。12億人へのOfficeライセンスは非現実的
- **EU**: デジタル主権政策。Microsoft依存からの脱却を推進
- **中国**: 米中対立でMicrosoft製品排除の動き

## 6.2 LibreOfficeの限界が明確になっている

30年経っても互換性が改善しない。特にアジア言語（日本語、Hindi、Tamil等）で致命的。
政府機関が「LibreOfficeで試したが諦めた」という事例が多数。

## 6.3 AI開発ツールによる開発速度の革命

従来：Office互換ソフトの開発には数百人×数年が必要
現在：**AI活用により1-2名×数ヶ月で同等品質を実現可能**

# 7. インド展開の提案

## なぜインドか

1. **言語問題が深刻**: Hindi、Tamil、Bengali等でWord→LibreOffice変換が壊滅的
2. **政府需要**: Digital India政策でOSS優先調達
3. **人材**: 世界最大のIT人材プール。サポートチーム構築に最適
4. **コスト**: インドでのチーム運営コストが低い

## 提案する体制

```
[日本] 開発本部（1-2名）        [インド] サポート・保守（5-10名）
 ├── コアエンジン開発             ├── テクニカルサポート
 ├── AI活用仕様解析              ├── 多言語対応（Hindi, Tamil等）
 └── 品質管理                     ├── カスタマイズ開発
                                  └── 政府・企業向け営業支援
```

# 8. 対象製品の拡張性

Oxiの技術基盤（Rust + WebAssembly）とAI仕様解析手法は、Office以外にも応用可能です。

| 対象ソフト | 市場規模 | 応用の方向性 |
|-----------|---------|-------------|
| Adobe Illustrator | 年間$2B+ | ベクター編集エンジン（oxivector） |
| Adobe Photoshop | 年間$5B+ | ラスター編集エンジン（oxiimage） |
| Adobe InDesign | 年間$1B+ | DTP組版エンジン（既に設計済み） |
| AutoCAD | 年間$5B+ | 2D/3D CADビューア |

**同じ開発手法（AI仕様解析 + Rust + WASM）で、上記すべてに横展開可能。**

人が増えれば、Microsoft + Adobeの市場（合計年間20兆円以上）を同時に攻略できます。
