再現精度は「主張」ではなく「実測」

以下の数値はすべて、リポジトリで公開されている再現可能な測定パイプラインによるものです。

Oxi のレイアウトエンジンは、実在する日本語ビジネス文書 235 件(410 ページ) — 就業規則・官公庁様式・契約書 — を対象に、Microsoft Word の出力とピクセル単位の SSIM(構造的類似度)で比較されています。2026-07-17 時点でページ平均 0.9378、文書平均 0.9591 です。

精度の根拠

  • 100% クリーンルーム実装: 逆アセンブルやバイナリ解析は一切なし。公開標準(OOXML — ISO/IEC 29500 / ECMA-376)と、COM API 経由で Word の出力値を観測するブラックボックス測定のみから仕様を導出。
  • デュアルフォントエンジン: Word のレイアウトは GDI の整数ピクセル丸めを引き継いでいるため、.docx では GDI を正確に再現。GDI のしがらみがない形式には DirectWrite の浮動小数メトリクスを使用。
  • 継続的な回帰ゲート: すべての変更を全コーパスで検証 — 改ページ一致・要素位置・ピクセル SSIM。

日本語組版をファーストクラスで

多くの Word 互換レンダラーで後回しにされてきた日本語レイアウトを、Oxi は実測対象の中心に据えています。

  • 禁則処理 — JIS X 4051 準拠の行分割: 行頭・行末禁則、約物の詰め、ぶら下げ組み。
  • 縦書き — 右から左へ流れる縦組みセクション、縦中横にも対応。
  • ルビ(ふりがな)、割注(行中 2 行の注)、圏点(傍点)。
  • 文字グリッド(docGrid) — Word の行グリッド・字送りモデルを忠実に再現。
  • 判子 — 丸印・角印・小判型のデジタル印鑑生成と PAdES PDF 署名。

測定コーパス自体が日本語中心です: 省庁の規程、行政の申請様式、雇用契約書 — 日本語レイアウトが最も難しい文書群で鍛えています。

技術

Rust + WebAssembly、形式非依存の中間表現、そしてサーバーゼロ。

なぜ今か

2026 年、フランス(DINUM 指令)・スイス(連邦事務局)・ドイツ(OpenDesk)など、欧州の公共部門は Microsoft Office 依存の解消を本格化させています。どの移行プロジェクトも同じ壁に当たります — 既存の .docx が「Word と同じ見た目」で開けること。Oxi は Word 再現度を約束ではなく継続測定で示すことで、この問題自体を解消することを目指しています。

アーキテクチャ

  • 完全クライアントサイド: 解析・レイアウト・描画・編集はブラウザ内 WebAssembly で実行。サーバー基盤もアップロードも不要。
  • 形式非依存の IR: Document → Page → Block (Paragraph | Table | Image) → Run。OOXML にも ODF にも依存しないため、往復変換で劣化しません。
  • ラウンドトリップ編集: 元の ZIP を保持し、変更された XML テキストノードだけをパッチ。ゼロから再構築しません。
  • オープンフォントのみ同梱: プロプライエタリフォントは同梱せず、メトリクス互換のオープンフォントを使用。
  • 組み込み可能: wasm-bindgen の JavaScript API(parse_document、layout_document、edit_docx、create_pdf など)を任意の Web アプリから呼び出せます。

ライセンス

コアエンジンは MPL-2.0(ファイル単位コピーレフト — AGPL と異なり商用製品へ組み込み可能)。WebAssembly / Python バインディングは MIT OR Apache-2.0。コントリビューションは DCO のみ、CLA 不要です。

よくある質問

Oxi は無料で使えますか?

はい。オープンソース(コア MPL-2.0、バインディング MIT OR Apache-2.0)で、Web デモはアカウント不要・無料です。

ファイルはサーバーにアップロードされますか?

いいえ。すべてブラウザ内の WebAssembly で処理され、ファイルは一切送信されません。機密文書でも安心して使えます。

Word とどれくらい同じ見た目になりますか?

実文書 235 件(410 ページ)で Word とピクセル単位の SSIM を継続測定しており、2026-07-17 時点でページ平均 0.9378 / 文書平均 0.9591。同一コーパスで LibreOffice・ONLYOFFICE を上回ります。再現精度セクションに比較表と測定手法があります。

対応形式は?

.docx(レイアウト+編集、中核ミッション)、.xlsx / .pptx(解析+編集、ビューア開発中)、PDF(解析・テキスト抽出・生成・署名検証)。

自社製品に組み込めますか?

組み込めます。MPL-2.0 のファイル単位コピーレフト+寛容ライセンスのバインディングにより、商用・プロプライエタリ製品への組み込みが可能です。

オフラインでも動きますか?

ページと WebAssembly モジュールの読み込み後は、文書処理はすべてローカルで完結します。ファイルを開く・描画する・編集するのにネットワーク接続は不要です。
